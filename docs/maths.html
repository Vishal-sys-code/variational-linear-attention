Math Primitives

This document outlines the core mathematical primitives implemented in src/maths/primitives.py. These primitives are rigorously designed for numerical stability and form the fundamental backbone of the Variational Linear Attention (VLA) system.

1. Inner-Product Score​

The attention score sts_tst​ at timestep ttt is computed as the scaled dot product of a key vector kt∈Rdk_t \in \mathbb{R}^dkt​∈Rd and a query vector qt∈Rdq_t \in \mathbb{R}^dqt​∈Rd:

st=kt⊤qtds_t = \frac{k_t^\top q_t}{\sqrt{d}}st​=d​kt⊤​qt​​

Implementation Details:

  • Returns: A scalar value indicating query-key relevance.
  • Scaling: Includes the standard 1/d1/\sqrt{d}1/d​ scaling factor to mathematically constrain gradients from vanishing or exploding, completely analogous to standard scaled dot-product attention.
  • Safety Measures: Validates that all values are strictly finite (void of NaN or Inf), which is crucially required before undertaking subsequent inverse operations.

2. Sherman–Morrison Rank-1 Inverse Update​

The core bottleneck of stably solving a linear system at every timestep is the matrix inversion. The Sherman-Morrison formula empowers VLA to surgically update the inverse of the penalty matrix MtM_tMt​ when it is mathematically perturbed by a rank-1 update utut⊤u_t u_t^\toput​ut⊤​.

Theoretical Foundation​

Given:

  • Mt−1∈Rd×dM_{t-1} \in \mathbb{R}^{d \times d}Mt−1​∈Rd×d: The prior penalty matrix (symmetric positive definite).
  • At−1=Mt−1−1A_{t-1} = M_{t-1}^{-1}At−1​=Mt−1−1​: The exact inverse of the prior penalty matrix.
  • ut∈Rdu_t \in \mathbb{R}^dut​∈Rd: The new update vector proposed by the PenaltyBuilder.

The updated penalty matrix naturally takes the form Mt=Mt−1+utut⊤M_t = M_{t-1} + u_t u_t^\topMt​=Mt−1​+ut​ut⊤​. Computing (Mt)−1(M_t)^{-1}(Mt​)−1 directly scales at O(d3)\mathcal{O}(d^3)O(d3). However, by leveraging the Sherman-Morrison identify, we reconstruct the inverse update securely in O(d2)\mathcal{O}(d^2)O(d2):

At=At−1−At−1utut⊤At−11+ut⊤At−1utA_t = A_{t-1} - \frac{A_{t-1} u_t u_t^\top A_{t-1}}{1 + u_t^\top A_{t-1} u_t}At​=At−1​−1+ut⊤​At−1​ut​At−1​ut​ut⊤​At−1​​

Algorithmic Execution Steps​

  1. Compute Denominator (Scalar):
δ=1+ut⊤(At−1ut)\delta = 1 + u_t^\top \left(A_{t-1} u_t\right)δ=1+ut⊤​(At−1​ut​)
  1. Numerical Safety Enforcement: If the absolute value ∣δ∣<ϵ|\delta| < \epsilon∣δ∣<ϵ, a strict fallback is triggered (adding ϵI\epsilon IϵI) stopping division by zero or gradient detonation. (Default ϵ≈10−6\epsilon \approx 10^{-6}ϵ≈10−6).
  2. Compute Intermediate Vector (z∈Rdz \in \mathbb{R}^dz∈Rd):
z=At−1utz = A_{t-1} u_tz=At−1​ut​
  1. Compute Outer Product (O∈Rd×dO \in \mathbb{R}^{d \times d}O∈Rd×d):
O=zz⊤O = z z^\topO=zz⊤
  1. Final Rank-1 Update:
At=At−1−OδA_t = A_{t-1} - \frac{O}{\delta}At​=At−1​−δO​
  1. Periodic Stabilization: Every KKK steps, add ϵI\epsilon IϵI onto the diagonal tensor of AtA_tAt​ to correct accumulating floating-point drift. This represents a critical fix for extremely long context sequence regimes.

3. Multiple Rank-1 Updates (Woodbury Generalization)​

To securely support higher-rank context parameterizations, we sequence iterating rank-1 updates {u1,u2,…,ur}\{u_1, u_2, \dots, u_r\}{u1​,u2​,…,ur​}. This explicitly mirrors the mathematical equivalence of the Woodbury matrix identity for a rank-rrr update—but iterating sequentially minimizes dangerous intermediate memory spikes.

A(0)=At−1A(i)=ShermanMorrison(A(i−1), ui)for i∈{1,…,r}At=A(r)\begin{aligned} A^{(0)} &= A_{t-1} \\ A^{(i)} &= \text{ShermanMorrison}\left(A^{(i-1)}, \ u_i\right) \quad \text{for } i \in \{1, \dots, r\} \\ A_t &= A^{(r)} \end{aligned}A(0)A(i)At​​=At−1​=ShermanMorrison(A(i−1), ui​)for i∈{1,…,r}=A(r)​

4. Recovering Optimal Coefficients α∗\alpha^*α∗​

In standard Linear Attention, the global memory matrix StS_tSt​ is linearly updated using static vtkt⊤v_t k_t^\topvt​kt⊤​. In sharp contrast, VLA computes a globally optimal scaling vector αt\alpha_tαt​ that completely minimizes the associative reconstruction error of the value vector vtv_tvt​ bounded strictly by the active penalty MtM_tMt​.

As VLA naturally tracks the inverted formulation At=Mt−1A_t = M_t^{-1}At​=Mt−1​, generating the theoretical ground-truth optimum α∗=Mt−1st\alpha^* = M_t^{-1} s_tα∗=Mt−1​st​ simplifies exponentially into a single matrix-vector hardware product:

αt=Atst\alpha_t = A_t s_tαt​=At​st​

5. Memory Matrix Update​

Having securely arrived at the optimal coefficient map αt\alpha_tαt​, the global memory matrix StS_tSt​ updates to ingest new contextual information:

St=St−1+αt⊗(vtkt⊤)S_t = S_{t-1} + \alpha_t \otimes \left(v_t k_t^\top\right)St​=St−1​+αt​⊗(vt​kt⊤​)

Implementation Details:

  • Batched Outer Products: Evaluated safely using v.unsqueeze(2) * alpha.unsqueeze(1) ensuring fast batched hardware execution without explicit loops.
  • In-place Constraints: The update strictly operates out-of-place (St=St−1+ΔS_t = S_{t-1} + \DeltaSt​=St−1​+Δ) guarding the PyTorch computational graph graph. (Direct memory overwrites like += are banned.)
  • Renormalization Guard: In regimes where the Frobenius norm ∥St∥F\|S_t\|_F∥St​∥F​ balloons over threshold, StS_tSt​ rescales safely avoiding numeric overflow during ultra-long runtimes.
