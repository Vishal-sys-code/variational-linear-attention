Penalty Builder Module

Overview​

The PenaltyBuilder is the central component that grants Variational Linear Attention its adaptive memory capabilities. Located in src/models/attention/penalty_builder.py, this module constructs the data-dependent penalty matrix Mt(θ)M_t(\theta)Mt​(θ) at each timestep ttt. By parameterizing MtM_tMt​, the model learns exactly how to weigh and forget historical information.

The module supports multiple parameterizations for the penalty matrix, balancing expressivity and computational cost.

Parameterizations​

1. Diagonal + Rank-1 (Mt=λtI+utut⊤M_t = \lambda_t I + u_t u_t^\topMt​=λt​I+ut​ut⊤​)​

The simplest and most robust parameterization. The matrix is decomposed into a uniform decay scalar (λt\lambda_tλt​) and a rank-1 outer product (utut⊤u_t u_t^\toput​ut⊤​), which allows the model to selectively penalize specific directions in the key-space.

Formulas:

λt=softplus(MLPλ(kt))\lambda_t = \text{softplus}(\text{MLP}_\lambda(k_t))λt​=softplus(MLPλ​(kt​)) ut=Wuktu_t = W_u k_tut​=Wu​kt​

Implementation Constraints:

  • λt\lambda_tλt​ is strictly clamped to ≥λmin\ge \lambda_{\text{min}}≥λmin​ (e.g., 10−310^{-3}10−3) to guarantee positive definiteness.
  • Wu∈Rd×dW_u \in \mathbb{R}^{d \times d}Wu​∈Rd×d is a learnable projection matrix mapping the key dimension to the update vector.
  • Output: Returns the tuple (\lambda_t, u_t).

2. Diagonal + Rank-r (Mt=λtI+∑m=1rut,mut,m⊤M_t = \lambda_t I + \sum_{m=1}^r u_{t,m} u_{t,m}^\topMt​=λt​I+∑m=1r​ut,m​ut,m⊤​)​

A higher-capacity parameterization that allows the model to penalize an rrr-dimensional subspace at each step.

Formulas:

λt=softplus(MLPλ(kt))\lambda_t = \text{softplus}(\text{MLP}_\lambda(k_t))λt​=softplus(MLPλ​(kt​)) ut,m=Wu,mktfor m=1,…,ru_{t,m} = W_{u,m} k_t \quad \text{for } m = 1, \dots, rut,m​=Wu,m​kt​for m=1,…,r

Implementation Constraints:

  • Ut=[ut,1,…,ut,r]U_t = [u_{t,1}, \dots, u_{t,r}]Ut​=[ut,1​,…,ut,r​] is computed efficiently via a single batched linear projection Wu∈R(r⋅d)×dW_u \in \mathbb{R}^{(r \cdot d) \times d}Wu​∈R(r⋅d)×d and subsequently reshaped into (B,r,d)(B, r, d)(B,r,d).
  • Output: Returns (\lambda_t, U_t).

3. Kernelized Penalty (Low-rank Approximation)​

This parameterization uses a generic feature map ϕ(x)\phi(x)ϕ(x) to approximate full kernelized penalty landscapes.

Formulas:

Mt≈λI+ϕtϕt⊤M_t \approx \lambda I + \phi_t \phi_t^\topMt​≈λI+ϕt​ϕt⊤​ ϕt=Wϕkt\phi_t = W_\phi k_tϕt​=Wϕ​kt​

Note: Currently implemented via KernelPenaltyBuilder. This shares the logic of Rank-1 but is designed for future expansions into full RBF kernel approximations.

Recurrence & Inverse Tracking​

The outputs from the PenaltyBuilder are not used as MtM_tMt​ directly; doing so would require O(d3)\mathcal{O}(d^3)O(d3) inversion. Instead, λt\lambda_tλt​ and utu_tut​ are passed to the InversePenaltyTracker module.

Crucial Detail: In VLALayer, the time-varying scalar λt\lambda_tλt​ from PenaltyBuilder is unused in the AtA_tAt​ update step. The recurrence strictly relies on utu_tut​ and an initial λ0\lambda_0λ0​. Consequently, the parameters within the λ\lambdaλ-network (lambda_net) do not receive gradients during the standard VLA forward pass.

Inputs and Outputs​

Inputs:

  • ktk_tkt​: The input key vector. Shape: (B, d) for streaming auto-regressive decoding, or (B, T, d) for batch-training.

Outputs:

  • λt\lambda_tλt​: Scalar base penalty. Shape: (B, 1) or (B, T, 1).
  • utu_tut​ (or UtU_tUt​): Update vector(s). Shape: (B, d) / (B, T, d) for rank-1, or (B, r, d) / (B, T, r, d) for rank-rrr.
  • stats: A dictionary containing internal tracking statistics (e.g., mean norms, eigenvalues) for logging via WandB or TensorBoard. Modules must return this dictionary rather than invoking logging frameworks directly.
